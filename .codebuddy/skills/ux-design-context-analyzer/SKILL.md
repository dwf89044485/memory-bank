# UX 设计上下文分析器

## 概述

此技能系统性地分析用户体验设计项目，将分散的设计材料、用户研究洞察和项目文档整合为结构化的 DESIGN-CONTEXT.md 文档，从而实现高效的 AI 协作。

## 使用场景

在以下情况调用此技能：
- 开始协作一个已有的设计项目
- 为新团队成员或 AI 助手梳理项目背景
- 在用户研究或设计阶段后整合项目知识
- 准备设计交付或重要项目里程碑
- 创建设计系统维护文档
- 为设计决策评审建立上下文

## 使用步骤

### 第一步：项目探索与分析

系统性地探索项目结构：

1. **识别设计工具和文件组织**
   - 定位 Figma、Sketch、XD 或其他设计文件
   - 梳理文件命名规范和版本控制方式
   - 记录设计系统或组件库位置

2. **收集用户研究材料**
   - 查找用户画像、用户旅程地图、研究报告
   - 识别可用性测试结果和用户反馈
   - 定位竞品分析和市场调研文档

3. **整理项目文档**
   - 定位产品需求文档（PRD）
   - 查找技术规格和约束条件
   - 识别利益相关者沟通记录

### 第二步：分析设计决策上下文

从已发现的材料中提取关键洞察：

1. **用户洞察综合**
   - 总结核心用户需求和痛点
   - 识别主要用户场景和边缘情况
   - 记录可访问性和包容性考虑

2. **设计约束梳理**
   - 技术限制和平台要求
   - 品牌指南和视觉识别约束
   - 业务目标和成功指标

3. **利益相关者对齐审查**
   - 决策流程和审批工作流
   - 关键利益相关者的关注点和优先级
   - 未解决的设计争议或待讨论问题

### 第三步：生成 DESIGN-CONTEXT.md

使用 `references/design-context-template.md` 中的模板创建文档，包含以下内容：

1. **项目概览**
   - 产品定位和价值主张
   - 目标用户群体及具体特征
   - 当前项目阶段和即将到来的里程碑
   - 关键成功指标和业务目标

2. **设计资产架构**
   - 文件组织体系和命名规范
   - 设计系统位置和使用指南
   - 组件库结构和维护流程
   - 版本控制和协作工作流

3. **用户研究洞察**
   - 主要用户画像及行为模式
   - 关键用户旅程和交互流程
   - 已验证的痛点和机会领域
   - 可访问性要求和考虑

4. **设计决策框架**
   - 已记录的设计原则及其依据
   - 技术和业务约束
   - 利益相关者达成的共识
   - 需要进一步验证的待定问题

5. **协作规范**
   - 设计评审流程和参与角色
   - 开发交付规格和工具
   - 反馈收集和迭代工作流
   - 沟通渠道和会议节奏

### 第四步：验证和完善上下文

生成初始 DESIGN-CONTEXT.md 后：

1. **与实际项目材料交叉核对**确保准确性
2. **识别信息缺口**，确定需要补充研究或文档的地方
3. **与团队成员或利益相关者验证**洞察的准确性
4. **随项目演进持续更新**

## AI 协助场景

创建 DESIGN-CONTEXT.md 后，AI 助手可有效支持：

- **设计评审准备**：基于项目上下文生成评审议程
- **用户研究分析**：结合已建立的用户画像综合分析发现
- **竞品分析**：研究与产品定位相关的竞争对手
- **内容策略**：创建符合用户心智模型的界面文案
- **可访问性审计**：根据记录的要求评估设计
- **设计系统更新**：维护与已建立模式的一致性
- **利益相关者沟通**：使用项目依据阐述设计决策

## 质量指南

确保 DESIGN-CONTEXT.md 保持：

- **具体性**：包含具体细节而非通用设计原则
- **可操作性**：聚焦于直接影响设计决策的信息
- **时效性**：定期更新以反映项目演进
- **完整性**：覆盖设计上下文的所有主要方面
- **清晰性**：使用清晰、无术语的语言，确保所有团队成员可理解
